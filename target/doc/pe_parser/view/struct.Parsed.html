pub struct Parsed<'a> { /* private fields */ }
Expand description

Wrapper around a parsed PE file, exposing higher-level helper methods. This allows transforming raw PE data into human-friendly representations.

pub fn new(raw: &'a PeFile) -> Self

Creates a new parsed wrapper instance from a raw PE file reference.

This enables access to structured views like pretty_json() or summary_json().

Source

pub fn pretty_json(&self) -> ParsedPretty

Builds a detailed, pretty-printed representation of the PE file.

Includes architecture, section info, entry point, characteristics, and optionally parsed import/export tables. Designed for human inspection.

Source

pub fn summary_json(&self) -> ParsedSummary

Returns a compact summary of key PE fields.

Intended for overview displays, such as CLI tools or file scanners.

Source

pub fn entry_point(&self) -> String

Returns the program entry point address in hexadecimal.

This is where execution begins when the program loads. The address is taken from the Optional Header.

Source

pub fn image_base(&self) -> String

Returns the image base address as a hexadecimal string.

This is the preferred virtual memory address where the image is loaded. It varies between PE32 and PE32+ formats.

Source

pub fn subsystem(&self) -> &str

Returns the subsystem type as a readable string.

Indicates the intended environment for the PE file (GUI, console, etc). Falls back to “Unknown number” if not recognized.

Source

pub fn dll_characteristics(&self) -> Vec<(u64, &str)>

Returns a list of DLL characteristic flags and their descriptions.

Flags are extracted from the Optional Header and filtered from known masks. Provides insight into binary security features like ASLR, DEP, etc.

Source

pub fn architecture(&self) -> &'static str

Identifies the PE machine architecture (e.g., x86, x64).

Reads the machine type directly from the PE header. Returns a string suitable for display in diagnostics or metadata.

Source

pub fn timestamp(&self) -> String

Converts the PE timestamp to a formatted UTC datetime string.

The timestamp comes from the FileHeader and represents the linker time. Useful for version comparison and forensic analysis.

Source

pub fn linker_version(&self) -> String

Returns the linker version in vX.Y format.

Source

pub fn os_version(&self) -> String

pub fn subsystem_version(&self) -> String

Source

pub fn characteristics(&self) -> Vec<(u16, &'static str)>

Extracts the COFF FileHeader flags and their meanings.

Flags describe attributes of the file (executable, stripped debug, etc). Filtered from a fixed known list.

Source

pub fn detect_type(path: &Path) -> Option<String>

Determines file type (EXE, DLL, etc.) based on file extension.

This is a best-effort heuristic based on common PE suffixes.

Source

pub fn sections(&self) -> Vec<ParsedSection>

Parses section headers and transforms them into structured, readable output.

Converts sizes and addresses into hex strings, and decodes section flags. Intended for visualization in tools or APIs.
